Notarization Evidence

This is a direct validation of transaction data concerning a notarization from a Quadrans Node.
Please note that Quadrans Foundation does not store a copy of the notarized data, nor has information about it's content or validity.
The following is an independent observation of the transaction data done for your convenience and can be done independently by any user

All the following steps make extensive use of the cryptographic hash function (SHA256).

Transaction data

Analizing the data in the provided transaction ${transactionHash}
with the matching contract ABI the following data was found:

Transaction data: 
transaction hash:${transactionHash}
block number:${blockNumber}
block hash:${blockHash}
block timestamp:${blockDateTime}
notarizazion sender:${sender}
smart contract address:${contract}
raw data:
${rawData}
Notarized data: 
file name:${_filename}
file hash:${_filehash}
notes: 
${_notes}

 

If you need further verification please use the smart contrac ABI to decode the transaction receipt and extract event logs and/or decode the hex data in the 'raw data' field into ASCII format to get the notarization parameters.

Check that the hash (SHA256) of the document ${_filename} is ${_filehash}.

If all those element match than you have solid proof that that :

  • the document was notarized on Quardans blockchain
  • by the owner of the wallet whose address is ${sender},
  • that the content of the file/data did not change over time,
  • and that it dates ${blockDateTime}.
    •

    You can refer to this proof either using this page address or by using the QR code:
    ${qr}

    Don't trust, verify.
